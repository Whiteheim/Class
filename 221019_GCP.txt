+ DBMS

1. 관계형 (Relational-Database)
1) 2차원 구조의 모델을 기반  ex) excel sheet
2) 데이터 무결성, 트랙젝션처리 등 데이터 베이스 
   관리 시스템에서 뛰어난 성능
3) 질의어(SQL)를 사용한 데이터 접근

2. 객체지향형 데이터베이스 (Object Oriented)
1) 사용자가 정의한 사용자 정의 타입을 지원.
2) 비정형 복합 정보의 모델링 가능
3) 객체들 사이의 참조(reference) 구조를 이용한 
    접근이 가능
4) 많은 객체지향 프로그래밍 언어들과 객체지향형 데이터베이스
   의 스키마 구조가 비슷.

3. 객체관계형 데이터베이스
 (Object-Relational DB)
1) 사용자 정의 타입을 지원
2) 참조 타입 지원
3) 중첩 테이블 지원
4) 대단위 객체 저장, 추출이 가능
5) 객체 간의 상속 관계를 지원하는 것이 가능.


+ BigQuery - 데이터 처리 아키텍처
1. RDBMS 같은 기능 (Ad-hoc query)
2. 맵리듀스 프레임워크 기능
3. 서버리스, 분산 SQL 엔진
4. ETL (Extract, Transform, Load)
         추출     변환          로드
5. 강력한 분석 도구
6. 시각화
